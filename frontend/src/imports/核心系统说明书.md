# 📖 语文一体化写作系统 - 核心系统说明书

## 1\. 项目概述与核心理念 (System Overview & Core Philosophy)

本系统是一个**AI驱动的个性化支架式教学平台**。其核心目标是将教师上传的静态教辅材料（PPT/PDF/Word），自动化地转化为“步进引导式 (Step-by-step)”的沉浸式互动学习网页。

**三大核心设计理念：**

1.  **以读促写 (Reading-to-Writing)：** 建立“感知与沉浸 $\rightarrow$技法拆解 $\rightarrow$片段练习 $\rightarrow$整体建构”的四阶段标准教学路径。
    
2.  **模式驱动界面 (Schema-Driven UI)：** 拒绝让 AI 直接编写前端代码（避免幻觉和崩溃）。AI 仅生成结构化的 JSON 协议，由预设的前端 React 组件库动态渲染页面。
    
3.  **人机协同工作流 (Human-in-the-Loop)：** AI 生成教学大纲后，流程挂起，强制允许教师在低代码界面进行预览和微调，确认无误后再发布给学生。
    

---

## 2\. 多智能体架构设计 (Multi-Agent Architecture)

后端的 AI 核心是由多个“专家智能体”组成的流水线（基于 LangGraph 编排）：

| 智能体角色 | 核心职责 | 技术实现重点 | 输出结果 |
| --- | --- | --- | --- |
| **1\. 课件解析专家 (The Reader)** | 理解上传文档，提取核心文本、知识点和教学目标。 | 结合 OCR 和视觉大模型理解图文排版。 | 结构化文本/知识点数据 |
| **2\. 教学设计专家 (The Architect)** | 将枯燥的知识点转化为“以读促写”四阶段互动剧本。 | 植入“支架式教学法”Prompt。 | 教学路径脚本 (逻辑步骤) |
| **3\. UI编排专家 (UI Orchestrator)** | 为教学步骤匹配最合适的前端交互组件（如填空、高亮、画板）。 | 严格遵守预设的 JSON Schema 协议。 | **Frontend Config JSON** (UI渲染协议) |
| **4\. 写作评测专家 (The Evaluator)** | 实时监听学生输入，结合当前任务上下文，给出伴学式反馈。 | 基于 WebSocket 的流式输出，携带 Prompt 上下文。 | 流式打字机反馈数据流 |

*(注：前端渲染引擎 The Builder 不是 AI，而是纯代码实现的 JSON 解析与组件映射器。)*

## 3\. 双端业务工作流 (Dual-End Workflow)

### 👨‍🏫 教师端工作流 (Teacher Workspace)

1.  **上传：** 上传课件 (PPT/PDF)。
    
2.  **AI 编排：** 后端 LangGraph 依次执行 `Reader -> Architect -> UI Orchestrator`。
    
3.  **图状态挂起 (关键)：** 生成初始 JSON 后，LangGraph 状态机暂停。
    
4.  **可视化微调 (低代码编辑器)：** 教师在左侧表单修改指令/字数/重点，右侧实时预览基于 JSON 渲染的互动网页。
    
5.  **确认发布：** 教师点击发布，后端固化 JSON，生成班级专属 Lesson Link。
    

### 👨‍🎓 学生端工作流 (Student Workspace)

1.  **进入课堂：** 打开 Link，前端读取 JSON 协议。
    
2.  **步进式学习：** 基于状态机，先渲染第一步（如阅读高亮）。学生完成后，平滑解锁第二步（如仿写）。
    
3.  **实时伴学：** 在需要打字或交互的组件中，防抖触发 `Evaluator Agent`，通过 WebSocket 获取针对性批改建议。

## 4\. 技术栈选型 (Tech Stack)

-   **前端框架:** Next.js (App Router) + React + Tailwind CSS + shadcn/ui
    
-   **前端状态管理:** Zustand (管理 Step 状态和表单数据) + React Query
    
-   **后端框架:** FastAPI (Python)
    
-   **AI 编排框架:** LangGraph (利用 `interrupt_before/after` 实现人类介入)
    
-   **数据库:** PostgreSQL (业务数据、JSON Schema 存储) + Redis (WebSocket 会话、LangGraph Checkpoint)
    
-   **双向通信:** WebSockets (专供 Evaluator Agent 实时反馈)

## 6\. 核心技术契约与实现规范 (Core Implementation Specs)

### 规范 1：Schema-Driven UI 前端渲染法则

前端**严禁**直接渲染 AI 生成的 HTML。必须实现一个 `SchemaRenderer`，根据 JSON 的 `type` 字段映射到预设的 React 组件。

-   左侧表单（教师端）修改 JSON 时，右侧预览区必须实现**实时热更新**。
    

### 规范 2：步进式渲染 (Step-by-step rendering)

学生端不能一次性展现所有内容。必须通过 Zustand 管理 `currentStep` 变量。

-   只有当当前组件抛出 `onComplete` 事件时，`currentStep` 才加 1，并平滑滚动加载下一个 JSON block。
    

### 规范 3：携带上下文的 WebSocket 通信 (针对 Evaluator)

学生在写作时，发往后端的 WebSocket Payload 必须包含**当前步骤的上下文**，以便 AI 知道“该用什么标准来评改”。

**数据契约示例：**

```json
{
  "student_id": "user_123",
  "lesson_id": "lesson_001",
  "current_step": 2,
  "component_type": "GuidedWritingArea",
  "student_text": "校园的早晨，雾气像...",
  "context": {
    "instruction": "请模仿上述句式描写晨曦", 
    "evaluator_focus": ["是否使用了比喻", "意境是否相符"] 
  }
}
```

## 7\. 给 AI 代码助手的开发指令 (Directives for AI Developer)

*当您（AI）阅读完上述文档后，请遵循以下原则协助我进行开发：*

1.  **自顶向下开发：** 请先帮我定义 `Schema` 契约（即生成一个完整的 `Lesson Schema JSON` 示例和后端的 `Pydantic Models`），这是前后端解耦的基石。
    
2.  **组件驱动：** 在编写前端页面时，大量使用 `shadcn/ui` 风格的原子组件来构建 `interactive-blocks`。
    
3.  **关注 LangGraph 状态机：** 在编写后端代码时，重点实现 `generator_graph.py` 中的 `StateGraph`，并正确使用中断机制 (`interrupt`) 以支持教师的 Human-in-the-loop 微调。
    
4.  **请回复“我已完全理解系统架构”**，并询问我希望从前端、后端还是数据协议哪一部分开始编写代码。
