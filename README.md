# 📚 Phase-1: 语文一体化课堂敏捷开发仓 (Demo)

## 一、 仓库描述 (Repository Description)

本仓库用于存放“AI驱动个性化支架式教学平台”第一阶段（Phase-1）的快速验证 Demo。
为了将开发周期极限压缩至一周内并快速响应校方需求，本项目采用 **AI 高度参与的敏捷开发模式**。摒弃传统重度工程化配置，以跑通核心教学场景（如材料解析、页面渲染、互动展示）的“主干道”为唯一优先级。

* **架构模式：** 轻量级 Monorepo（单体仓库），前后端代码同仓管理，减少 AI 跨仓库理解上下文的成本。
* **技术栈约束：**
* **UI 与前端骨架：** Figma + Figma Make (一键生成核心 UI 代码)
* **前端逻辑与细节：** Claude (负责组件状态、小需求及 API 对接)
* **后端支撑：** Claude (全量生成，提供 RESTful API 或直接使用轻量级 Serverless/BaaS 方案，视情况提供 Mock 数据)



## 二、 系统描述 (System Description)

本 Demo 系统是一个由教辅材料（假定已解析）驱动的强交互 Web 教学应用。在阶段一，我们暂不追求底层的“动态 JSON 渲染引擎”完美闭环，而是通过硬编码或半自动的方式，快速呈现最终的产品形态。

* **前端 (Frontend - View & Interaction)：**
直接承接由 Figma Make 导出的 React/Vue 组件。系统会展示一套固定的教学流模板，包含核心的交互组件（如阅读器、批注区、简单的互动问答 UI）。前端逻辑极简，重展示，轻状态管理。
* **后端 (Backend - API & Mock)：**
提供为前端定制的“保姆式 API”。由于是 Demo 阶段，后端主要承担数据组装和分发的角色。如果涉及复杂的 AI 解析（如 PDF 提炼），此阶段采用预先准备好的 Mock 数据（即人造的理想 JSON 结果）进行接口返回，确保演示时 100% 稳定不死机。
* **核心准则：** “视觉即产品”。只要界面上的“一键生成”按钮按下去，能丝滑展现出教学内容的转化结果，Demo 的使命就完成了。

## 三、 开发流程规范 (Workflow Specification)

这是我们与 AI（Claude、Figma Make）协同作战的流水线 SOP。严格遵守，避免 AI 产生代码幻觉导致代码库崩塌。

### 1. 需求与设计流 (UI -> Code)

* **Figma 设计：** 在 Figma 中完成页面设计，**必须**严格规范图层命名（例如 `ReadComponent`, `UploadButton`），方便生成语义化代码。
* **Figma Make 生成：** 使用插件一键导出前端骨架代码（HTML/CSS 或基础框架代码）。
* **入库：** 将生成的纯静态代码作为一个独立的 Commit 提交，附带备注 `chore: Figma Make 初始生成 [页面名称]`。

### 2. 前端加工流 (Claude Refinement)

* **上下文投喂：** 将 Figma 生成的代码丢给 Claude，并附带明确的小需求指令（例如：“帮我把这个静态按钮加上点击 loading 状态，并预留一个发请求的 onClick 函数”）。
* **原子化微调：** **严禁**让 Claude 一次性重构整个页面。小需求、小范围替换。
* **测试与提审：** 替换代码后，本地跑通即可提交。

### 3. 后端生成流 (Claude Backend)

* **API 契约先行：** 在让 Claude 写后端逻辑前，先让它根据前端页面输出一份接口文档（Markdown 格式）。
* **代码生成：** 确认接口无误后，让 Claude 吐出完整的 Controller 和 Service 层代码。
* **防御性编程：** 提示词中必须强制要求 Claude 增加 `try-catch` 和容错日志，防止 Demo 演示时直接白屏。

### 4. Git 极简分支与提交规范

由于是快速出 Demo，我们采用最轻量的分支策略：

* `main` 分支：永远保持可运行的演示版本。**不可直接在此分支开发**。
* `dev` 分支：主开发分支。跑通即合并。
* **Commit 规范（带 AI 标识）：** 为了复盘方便，建议在 Commit 加上 AI 辅助的标记。
* `feat(ui): 🤖[Figma] 生成阅读批注界面骨架`
* `fix(frontend): 🤖[Claude] 修复录音组件状态不同步问题`
* `feat(api): 🤖[Claude] 新增课时 JSON 获取接口并使用 mock 数据`
